# Security Policy

## Reporting A Vulnerability

Do not open a public issue for a vulnerability that could expose user data, secrets, or an authorization bypass.

Until a dedicated security inbox is configured, contact the repository owner privately through the email address on their GitHub profile. Include:

- The affected TenantProof version
- Reproduction steps
- The expected and observed behavior
- Whether the issue could cause a false pass

## Scope

TenantProof is an early-stage audit tool. Treat its findings as supporting evidence, not as a substitute for professional security review. Run mutation checks only against disposable local or staging projects without production customer data.
